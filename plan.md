## 1. 项目目标与成功指标

### 1.1 商业目标（按优先级）

1. **主目标｜询盘获取**：让日本品牌方、商社、买手、独立品牌主理人在搜到/访问网站后，能在 5 分钟内决定"想跟万翔辉聊一聊"。
2. **次目标｜品牌信任建设**：把"中国工厂"标签转化为"中国 + 缅甸双引擎、日本品质标准、20+ 年实绩"的可信信誉资产。
3. **次目标｜内容 SEO 资产**：在日文长尾关键词上（リュック OEM 中国 / バッグ OEM 海外 / 日本品質 中国工場 等）建立 30+ 篇优质内容，长期吸引询盘。
4. **辅目标｜招聘 / PR 入口**：作为 PR 报道、媒体采访、招聘的统一入口。

### 1.3 非目标（明确不做的事）

- ❌ 不做 DTC 消费者电商（不挂购物车、不挂购买按钮）
- ❌ 不做面向中国国内市场的展示（中文版仅作为内部辅助 / 来访客户参考）
- ❌ 不做万能贸易公司形象（不展示与箱包无关的业务）
- ❌ 不做"硬中式风格"（不用红金大花纹、不放大幅厂房航拍 + 标语横幅）

## 2. 目标用户与使用场景

### 2.1 主要用户画像（Persona）

#### Persona A：日本商社 / 大型零售采购员
- 年龄 35-50，男性居多，西装、保守
- 工作流：上司给一个产品 brief → 找 3-5 家工厂询比价 → 索取 PDF カタログ → 拜访工厂 → 决定
- **关心**：起订量、纳期、交付稳定性、合规性、过往实绩、付款条件、出口经验
- **入口**：Google 搜索 + 行业展会名片 + 现有合作商社介绍
- **决策周期**：2-6 个月

#### Persona B：日本中型品牌方 / EC 品牌
- 年龄 28-45，男女皆有，时尚行业属性
- 工作流：有产品概念草图 → 找能小批量打样的工厂 → 反复打样 → 量产
- **关心**：打样速度、设计配合度、面料/工艺可达性、小批量是否接、沟通效率（语言）
- **入口**：Google 搜索 + Instagram + 行业 KOL 推荐
- **决策周期**：1-3 个月

#### Persona C：日本独立品牌主理人 / 设计师
- 年龄 28-40，注重"故事和工艺"
- 工作流：找一个能合作长期、互相讲得来的工厂
- **关心**：工厂的"人味"、博客活跃度、是否有理念契合的实绩
- **入口**：日文 SEO 内容、长尾文章、社交分享
- **决策周期**：3-6 个月

#### Persona D：日本贸易商 / 既存伙伴
- 已是客户的"老熟人"
- 来网站找特定资料：PDF カタログ、最新动态、联系入口、新工厂介绍
- **关心**：信息更新、版本管理、文件下载便利

### 5.2 当社の強み（/strength）

#### 页面目的
把首页第 3 屏的"5 つの強み"展开为详细论证，让 B2B 访客在此页停留 2-3 分钟后产生"这家不一样"的感受。

#### 5 大强项详细写法

##### 强项 1：中国 + 緬甸 双拠点（核心差异化）

- **标题**：「中国と緬甸、二つの自社工場」
- **副标题**：「同じ会社、二つの強みを使い分ける。」
- **内容**：
  - 中国山东（日照）工厂：8,000 ㎡ / 约 100 名 / 平车 120 台 / 电脑车 22 台 / 适合打样、急单、多款小批量；
  - 缅甸仰光工厂（Better Bags Myanmar）：约 370 名 / 平车 420 台 / 电脑车 48 台 / 高车 60 台 / 双针车 12 台 / 适合中大货长单（单款单色 1,500 件以上）；
  - 「使い分けの考え方」表格：客户场景 → 推荐工厂 → 起订量 → 纳期。
- **视觉**：双工厂照片左右并置 + 一张地图（青岛 / 日照 / 仰光三点连线）
- **数据来源**：知识库 02、03

##### 强项 2：青島打样中心 1 周首样

- **标题**：「青島打样中心、最短 1 週間でサンプルアップ」
- **内容**：
  - 排期 + 制作的 "排期+制作 = 总周期" 透明化说明；
  - 接受 1-2 次合理修改的标准流程；
  - 4 次以上反复打样的"红线"政策（用更柔和的客户视角表达："デザインの方向性をご決定いただいた上でご依頼ください"）。
- **数据来源**：知识库 05

##### 强项 3：二段検品 + X 線検針機

## 10. 技术栈与架构

### 10.1 推荐技术栈

#### 选项 A：Headless 现代化（推荐）
- **前端**：Next.js 14+（App Router） / TypeScript / Tailwind CSS
- **CMS**：Strapi（自托管） 或 Sanity / Contentful（SaaS）
- **图床/CDN**：Cloudinary / Vercel Image Optimization
- **托管**：Vercel（亚洲节点 / 包括日本 Tokyo edge） 或 Cloudflare Pages
- **数据库**：PostgreSQL（CMS 自带）
- **表单后端**：Resend / SendGrid（邮件）+ Notion API / Airtable API（数据落库）/ HubSpot CRM
- **分析**：Google Analytics 4 + GTM + Microsoft Clarity（行为录屏）
- **搜索**：Algolia（如内容多）/ 站内搜索可选

**优势**：性能极佳、SEO 友好、未来可扩展、TypeScript 类型安全
**劣势**：开发门槛较高、需要前端 + 后端 + CMS 三层维护

#### 选项 B：WordPress（保守选项）
- **CMS**：WordPress + 日文/英文/中文 多语言插件（WPML 或 Polylang）
- **主题**：定制开发或基于 Sage/Roots 框架
- **托管**：Kinsta / WP Engine / SiteGround 日本节点

**优势**：易学易维护、生态丰富、文章发布门槛低
**劣势**：性能调优麻烦、安全维护成本高、品牌质感天花板低

#### 选项 C：Astro（极致性能选项）
- **前端**：Astro 4+ / TypeScript / Tailwind
- **CMS**：Sanity / Decap CMS / 文件型 markdown
- **托管**：同 A

**优势**：性能极致（静态 HTML 为主）、SEO 极强、轻量
**劣势**：动态交互能力较弱、生态较新

#### 建议：**首选 A（Next.js + Strapi/Sanity）**
- 与现代日式 B2B 网站气质（KT-Pack、榮伸 等）契合
- 内容更新便利
- 未来上线 EC、Customer Portal、内部系统都可扩展
